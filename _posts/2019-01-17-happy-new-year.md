## Happy New Year

A new year, a new blog then!

I've been a long-time WordPress user, previously writing at [kzhiwei.wordpress.com](https://kzhiwei.wordpress.com/).

My current workflow is:

1. Create/edit drafts locally using a good text editor;
1. Copy-paste into a new WordPress post, and then hit Publish.

In other words, I don't use WordPress as intended: writing, editing and publishing directly in the WordPress UI proper. Plus I do sometimes format text in __bold__, _italic_, or even `code`. All of which doesn't carry over when I copy/paste, which only serves to increase friction.

I guess it was inevitable that I'd migrate. Of late I've been using [Markdown](https://daringfireball.net/projects/markdown/syntax) to write technical documentation. It's like a simplified version of HTML.

HTML is... too verbose. There are opening tags and closing tags, it just gets in the way of producing actual content.

So I've known about [GitHub Pages](https://pages.github.com/) for awhile now, but this is my first time actually using it. I'm just going to `git add; git commit; git push`, see how it turns out then!
